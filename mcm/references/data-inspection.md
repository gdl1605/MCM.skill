# Data Inspection

服务 `SKILL.md` Step 1、Step 4、Step 6：拿到数据后先做赛时初检，避免主键、粒度、时间轴、proxy target 或高风险结构误判。

最应该调用的场景：已经读入附件/表格，准备建模、预处理、锁 schema 或套用 snippet 之前。

证据等级：[高频共性]
证据来源：`references/distill/modeling-playbook.md` 3.1/3.2；`references/distill/evidence-map.md` 中 2021/2023/2025 的粒度、结构性 0、多时点与 proxy target 证据；Phase 8B/8C 回归路由评测总结。

## 硬规则

如果 60 秒内说不清“主键、粒度、时间轴、目标来源、最终表头”，不要继续深挖模型。先完成初检记录。

## 1. 60 秒数据初检清单

| 检查项 | 必看内容 | 发现问题后怎么处理 |
| --- | --- | --- |
| 字段是否齐全 | 题目要求字段、附件说明字段、目标列、约束列是否存在 | 缺字段时先回题面确认是否由其他表计算，不要直接建模 |
| 缺失值 | 每列缺失率、关键列缺失、缺失是否集中在某类对象/时间 | 区分真实缺失、未采集、结构性空白；关键列缺失要记录修补规则 |
| 重复记录 | 全行重复、主键重复、同一对象同一时间多行 | 判断是脏数据、重复测量还是多明细粒度，不能直接去重 |
| 粒度不一致 | 原始流水、日汇总、对象汇总、品类汇总是否混在一起 | 先锁建模粒度和最终答卷粒度，再聚合 |
| 单位/量纲混乱 | kg/g、元/万元、百分比/小数、日期格式 | 先统一单位并写进处理日志 |
| 时间字段 | 日期能否解析、是否连续、是否有预测/决策时间窗 | 时间窗不清时不要套预测或优化 snippet |
| 对象字段 | 人/物/品类/地区/设备等对象 ID 是否能唯一定位一行 | 主键不清时先建字段字典和候选主键表 |

## 2. 高风险结构快速识别

| 结构 | 识别信号 | 立即动作 |
| --- | --- | --- |
| 重复测量 / 同一对象多次观测 | 同一对象在不同时间、批次、环境下多行 | 标记为“非独立行”，至少生成对象级汇总和时间级轨迹 |
| 多时点追踪 | 有连续或离散时间点，问题关心变化、趋势、达标过程 | 先按对象 × 时间排序，不要把每行当独立样本 |
| proxy target / surrogate criterion | 目标列是检测标签、销量、评分、专家规则、替代指标 | 在变量表里写清“代理了什么”，结论保持保守 |
| 阈值达标时间 / 首次 crossing | 题目问“何时达到”“最早满足”“首次超过/低于” | 转为 first-hit/达标率表；不要只做普通点预测 |
| 分阶段筛选 / 多层判定 | 先筛候选，再排序/优化/判定 | 每阶段保存候选表，避免最终推荐无法追溯 |

## 3. 发现后必须回退的情况

### 必须回到题型判断

- 发现题目问的是“首次达标时间”，但原计划按普通预测处理。
- 发现标签是 proxy，而原计划把它当真实标签训练分类器。
- 发现同一对象多次观测，但原计划把每行当独立样本。
- 发现开放问答实际要求推荐/判定表，而原计划只写文字分析。

### 必须先锁 schema

- 题目有 workbook、附件矩阵、命名结果文件或固定答案表。
- 最终输出对象是 `日期 × 对象`、`对象 × 周期`、`候选项 × 推荐值` 这类明确粒度表。
- 上游输出要喂给下游优化/分类/写作，列名不稳定会拖垮后续。

### 不能继续深挖模型

- 主键或时间轴未定义。
- 单位/量纲明显冲突。
- 目标列来源不明，且无法说明是 proxy 还是 true target。
- 最终答卷表的对象列和结论列还没有确定。
