# 摘要素材池模板

证据等级：[高频共性]
主要证据来源：`references/distill/writing-playbook.md` 摘要公式；`references/distill/evidence-map.md` 中“任务-方法-结果-特点”摘要组织规律。

## 使用规则

- 结果开始稳定时就填，不要等最后才想摘要。
- 每栏至少写一句可放入摘要的中文素材。
- 结果栏必须有具体结果、表名或结论对象，不能只写“效果较好”。

| 维度 | 最少素材要求 | 填写内容 |
| --- | --- | --- |
| 背景 | 题目场景、总体任务、关键难点 |  |
| 方法 | 按问题或模块写明模型/规则及作用 |  |
| 结果 | 写最关键的排序、预测、方案、判定或推荐结果 |  |
| 特点 | 写稳健性、可解释性、交付性或应用价值 |  |

## 最小完成标准

- 背景栏能说明“为什么要建模”。
- 方法栏能对应每个主要问题。
- 结果栏至少出现一个具体 final answer artifact。
- 特点栏不能空泛，必须对应验证、可解释性或可交付表格。
