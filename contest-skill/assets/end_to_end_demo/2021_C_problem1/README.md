# 2021 C 题问题一最小贯通 demo

证据等级：[中频经验]
主要证据来源：`references/distill/evidence-map.md` 中 2021-C066、2021-C169、2021-C283；2021 mini blind run 与 answer-sheet check 对 Top50 名单表的要求。

## 适用场景

当问题要求从候选供应商/对象中筛选一批优先对象时，先打通这条最小链路：

`题面目标 -> 特征表 -> 熵权 TOPSIS 排序 -> Top-k 结果表 -> 正文结论句`

## 非目标

- 不是完整 2021 C 题全题解。
- 不生成 24 周订购矩阵或转运矩阵。
- 不复刻任一优秀论文的完整模型。
- 不替代正式数据清洗和参数校准。

## 输入说明

示例输入文件：`sample_suppliers.csv`

最小字段：

| 字段 | 含义 | 方向 |
| --- | --- | --- |
| `supplier` | 供应商编号 | 主键 |
| `total_supply` | 供货能力或历史供货量 | 正向 |
| `stability_rate` | 稳定性指标 | 正向 |
| `material_importance` | 材料重要性/供给价值 | 正向 |
| `loss_rate` | 损耗或违约风险 | 负向 |

## 模型步骤

1. 读取候选供应商特征表。
2. 对正向/负向指标做同向化和标准化。
3. 用熵权法生成指标权重。
4. 用 TOPSIS 得到综合得分和排序。
5. 导出 `Top50 供应商表` 的示例结果。

## 运行方式

在项目根目录执行：

```bash
python3 assets/end_to_end_demo/2021_C_problem1/run_demo.py
```

运行后会生成：

- `assets/end_to_end_demo/2021_C_problem1/outputs/top50_suppliers_demo.csv`

## 输出结果表说明

| 字段 | 含义 | 正文用途 |
| --- | --- | --- |
| `supplier` | 供应商编号 | 说明最终入选对象 |
| `score` | 综合评价得分 | 支撑排序依据 |
| `rank` | 排名 | 形成 Top-k 名单 |
| `weight_*` | 指标权重 | 解释模型可解释性 |

## 验证/核查

至少做两类核查：

1. 权重扰动：将熵权替换为等权，检查前若干名是否稳定。
2. 指标删减：去掉一个风险或规模指标，检查 Top-k 是否大幅变化。

## 可直接转写的结论句

针对问题一，我们以供货能力、供货稳定性、材料重要性和损耗风险构建供应商综合评价指标，并采用熵权 TOPSIS 得到候选供应商得分。排序结果显示，综合得分靠前的供应商在供货规模和稳定性上表现更优，可作为后续订购与转运优化的优先候选集，完整名单见 Top50 供应商表。
