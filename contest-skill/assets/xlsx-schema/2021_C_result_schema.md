# 2021 C 题结果 schema 锚点

适用场景：2021 C 类供应商筛选、订购计划、转运计划、产能提升类答卷表设计。
来源锚点：2021 答案表检查、2021 mini blind run；`references/distill/evidence-map.md` 中 2021-C066 等优秀论文摘要证据。
证据等级：[中频经验]

## 非目标

- 不是正式 Excel 成品。
- 不填满 24 周真实数值。
- 不要求所有优化题都使用 24 周矩阵。

## 结果对象 1：Top50 供应商表

| 列名 | 含义 | 是否为最终答卷支撑列 | 正文呼应方式 |
| --- | --- | --- | --- |
| `rank` | 综合排名 | 是 | “排名前 50 的供应商见表 x” |
| `supplier_id` | 供应商编号 | 是 | 指明入选对象 |
| `score` | 综合得分 | 是 | 支撑排序依据 |
| `main_material` | 主要材料类别 | 按需 | 解释后续产能/订购约束 |
| `supply_capacity_index` | 供货能力指标 | 是 | 说明规模优势 |
| `stability_index` | 稳定性指标 | 是 | 说明可靠性 |
| `risk_index` | 损耗/违约风险指标 | 是 | 说明风险控制 |

## 结果对象 2：24 周订购矩阵

| 列名/维度 | 含义 | 是否为最终答卷支撑列 | 正文呼应方式 |
| --- | --- | --- | --- |
| 行：`supplier_id` | 被选供应商 | 是 | 指出每个供应商计划 |
| 列：`week_1` 至 `week_24` | 每周订购量 | 是 | 对应官方模板/附件 |
| `total_order` | 24 周总订购量 | 是 | 正文概括规模 |
| `material_type` | 材料类别 | 按需 | 对应产能折算 |

## 结果对象 3：24 周转运矩阵

| 列名/维度 | 含义 | 是否为最终答卷支撑列 | 正文呼应方式 |
| --- | --- | --- | --- |
| 行：`transporter_id` 或 `supplier_id` | 承运商/供应商 | 是 | 指出运输分配 |
| 列：`week_1` 至 `week_24` | 每周转运量 | 是 | 对应转运方案 |
| `loss_rate_used` | 使用的损耗率 | 是 | 解释转运选择 |
| `capacity_used` | 使用能力/比例 | 按需 | 说明可行性 |

## 结果对象 4：方案效果表

| 列名 | 含义 | 是否为最终答卷支撑列 | 正文呼应方式 |
| --- | --- | --- | --- |
| `scenario` | 方案名称 | 是 | 对比不同方案 |
| `objective_value` | 目标函数值 | 是 | 说明优化收益 |
| `total_supply` | 总供给量 | 是 | 说明供给保障 |
| `total_loss` | 总损耗 | 是 | 说明风险/成本 |
| `feasible` | 是否满足约束 | 是 | 支撑可行性判断 |
| `note` | 备注 | 按需 | 写明特殊约束或假设 |

## 使用规则

1. 先锁行列粒度，再写模型。
2. 正文引用时必须说明结果表回答了哪一问。
3. 模板型结果保留完整矩阵，正文只放摘要表或关键周示例。
4. 最终答卷支撑列不得只存在于代码变量中，必须导出为表。
