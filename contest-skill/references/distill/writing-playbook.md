# Phase 1 写作 Playbook

## 使用方式

本文件服务于后续 `contest-skill` 的写作部分，不追求“论文欣赏”，只保留比赛中最可复用的写法规则。证据强度同样分为：

- `[高频共性]`
- `[中频经验]`
- `[低频个例]`
- `[弱证据]`

## 1. 标准论文结构模板

## 1.1 近五年高频稳定结构

| 章节 | 常见程度 | 典型作用 | 建议篇幅 |
| --- | --- | --- | --- |
| 摘要 | 几乎必有 | 交代每问方法、结果、特色 | 0.5-1 页 |
| 问题重述 | 几乎必有 | 压缩题意，建立后文变量与任务边界 | 1-2 页 |
| 问题分析 | 几乎必有 | 解释每问为什么这样建模，搭起题间依赖 | 1-3 页 |
| 模型假设 | 高频 | 去噪、限定适用边界，为后文约束/模型服务 | 0.5-1 页 |
| 符号说明 | 高频 | 控制记号混乱，降低正文理解门槛 | 0.5-1 页 |
| 数据预处理/数据说明 | 高频 | 说明数据来源、清洗、聚合、特征构造 | 1-2 页 |
| 各问题模型建立与求解 | 核心主体 | 正文主战场，按问题依次展开 | 论文的 50%-65% |
| 结果分析 | 高频 | 把数值、图表、方案翻译成题目语言 | 1-3 页 |
| 模型评价与推广 | 高频 | 写优缺点、稳健性、适用范围与改进方向 | 1-2 页 |
| 结论 | 高频 | 总结每问回答，不重复摘要 | 0.5-1 页 |
| 参考文献 | 必有 | 保持格式规范 | 适量 |
| 附录/代码 | 高频 | 放推导、代码、补充图表，避免正文过载 | 视需要 |

## 1.2 哪些章节最稳定

- `摘要 -> 问题重述 -> 问题分析 -> 模型建立与求解 -> 结果分析 -> 模型评价` 是最稳定主线。 `[高频共性]`
- 对数据较重的题目，优秀论文常把“数据预处理”单列或嵌入问题分析之后。 `[高频共性]`
- 对组合题，正文主体通常按“问题一、问题二、问题三……”分段，每问内部再写“建模 -> 求解 -> 结果”。 `[高频共性]`

## 1.3 哪些章节可以灵活调整

- 模型假设和符号说明可前置，也可和问题一建模部分合并。 `[中频经验]`
- 结果分析可放在每问末尾，也可集中成一节，但高分论文更常见的是“每问即刻回题”。 `[中频经验]`
- 数据预处理可以拆成独立章节，也可以分散到每问中，但粒度一定要说明清楚。 `[高频共性]`

## 2. 摘要公式

## 2.1 最稳定的四段式

**模板顺序** `[高频共性]`

1. **背景/任务重述**
   - 用 1-2 句说明题目场景和总体目标。
2. **方法概述**
   - 按题目顺序或按模块顺序交代每问主模型。
3. **关键结果**
   - 给出最有代表性的数值、策略、分类结果、收益变化或显著性结果。
4. **模型特色或优点**
   - 用 1 句交代稳健性、适用性、创新点或泛化价值。

## 2.2 有效写法

- 不只报模型名，要说明“模型用来解决哪一问的什么任务”。 `[高频共性]`
- 关键结果尽量落到数字、比例、改善幅度、分组边界。 `[高频共性]`
- 结尾宜点出稳健性、可解释性或应用价值，而不是空喊“具有现实意义”。 `[高频共性]`

## 2.3 常见空泛写法

- 只把题目重述一遍，不说解法。
- 连续堆 5-6 个模型名，却不说明各自对应哪一问。
- 只写“模型效果较好”“结果合理”，没有任何结果支撑。

## 2.4 按题型的细微差异

- 优化题摘要要写出“方案效果”，例如收益、成本、损耗、产能或风险变化。 `[高频共性]`
- 分类/判别题摘要要写出“判定依据或分类效果”，例如特征筛选、准确率、显著性。 `[中频经验]`
- 机制解释题摘要要写出“关系方向与检验结果”，而不是只说“建立了关系模型”。 `[中频经验]`

## 3. 各章节写法模板

## 3.1 问题重述

- **作用**
  - 把题面压缩成后文建模能直接接住的任务描述。
- **最常见结构**
  - 场景一句话 -> 数据来源一句话 -> 按问题列出目标。
- **容易写空的地方**
  - 大段抄题面背景，不做压缩。
- **优秀论文的做法**
  - 直接重写成“对象-目标-约束-输出”的语言，方便后文承接。 `[高频共性]`

## 3.2 问题分析

- **作用**
  - 解释为什么某一问属于评价、预测、优化或分类；说明题间依赖。
- **最常见结构**
  - 先拆每问任务，再给建模路线图。
- **容易写空的地方**
  - 只写“问题一比较简单，问题二较复杂”之类套话。
- **优秀论文的做法**
  - 明确写出“先做什么、再做什么、为什么这样串联”。 `[高频共性]`

## 3.3 模型假设

- **作用**
  - 把题目中的模糊条件变成可计算前提。
- **最常见结构**
  - 逐条列假设，并让每条假设对应后文一类约束或变量处理。
- **容易写空的地方**
  - 写“忽略外界影响”“假设数据真实”这种与后文无关的空假设。
- **优秀论文的做法**
  - 假设数量不多，但每条都能在后文找到落点。 `[高频共性]`

## 3.4 符号说明

- **作用**
  - 控制变量和参数混乱，减少后文公式解释成本。
- **最常见结构**
  - 只列核心决策变量、参数、集合和指标。
- **容易写空的地方**
  - 把全文所有出现过的字母都堆进表里，读者反而找不到重点。
- **优秀论文的做法**
  - 只保留会反复出现的核心记号，并保持全文一致。 `[高频共性]`

## 3.5 模型建立与求解

- **作用**
  - 正式回答每个子问题。
- **最常见结构**
  - 变量定义 -> 目标函数/判别准则 -> 约束/关系式 -> 求解方法 -> 结果解释。
- **容易写空的地方**
  - 一上来堆公式，没有说明这些变量和题意怎么对应。
- **优秀论文的做法**
  - 每引入一个模型，都先写“它要解决什么”，再写公式。 `[高频共性]`

## 3.6 结果分析

- **作用**
  - 把图表和结果值翻译成题目语言。
- **最常见结构**
  - 先引用图表，再解释规律，最后回题。
- **容易写空的地方**
  - 只报数不解释，或只写“从图中可以看出”但没说看出了什么。
- **优秀论文的做法**
  - 一段只讲一个判断，例如“谁最重要”“哪个方案更稳”“哪些参数最敏感”。 `[高频共性]`

## 3.7 模型评价与推广

- **作用**
  - 补足可信度、适用范围和改进方向。
- **最常见结构**
  - 优点 2-3 条 -> 局限 2-3 条 -> 推广/改进建议。
- **容易写空的地方**
  - 只写“模型简洁、应用广泛、具有现实意义”。
- **优秀论文的做法**
  - 优缺点都紧贴本题的数据条件、约束结构和验证结果。 `[高频共性]`

## 3.8 结论

- **作用**
  - 用最短篇幅回收全文。
- **最常见结构**
  - 按问题顺序总结回答，再补一句整体建议。
- **容易写空的地方**
  - 把摘要换个说法重写一遍。
- **优秀论文的做法**
  - 明确写出“问题一得到什么，问题二给出什么策略，问题三说明什么”。 `[高频共性]`

## 4. 高频句式与过渡方式

以下不要求逐字照搬，重点是学习表达模式。

## 4.1 问题分析过渡句

- `[高频共性]` “问题一的核心在于量化……，因此先从……特征出发构建评价指标体系。”
- `[高频共性]` “由于问题二的求解依赖问题一的筛选结果，故以下在候选集确定后建立……模型。”
- `[中频经验]` “该问题本质上是一个同时含有……与……约束的优化决策问题。”

## 4.2 模型引入句

- `[高频共性]` “为刻画……之间的关系，建立……模型。”
- `[高频共性]` “在满足……约束的前提下，以……为目标函数进行优化。”
- `[中频经验]` “考虑到数据存在……特征，采用……可以更好地处理……问题。”

## 4.3 结果描述句

- `[高频共性]` “结果表明，……较基准方案……，说明……”
- `[高频共性]` “由表 x 可知，排名前列的对象在……指标上表现更稳定。”
- `[中频经验]` “模型拟合指标显示……，说明当前模型能够较好反映……”

## 4.4 对比分析句

- `[高频共性]` “与未考虑……的情形相比，考虑……后……”
- `[高频共性]` “在两种销售假设下，方案表现存在明显差异，主要体现在……”
- `[中频经验]` “相较于固定效应模型，混合效应模型在……指标上更优。”

## 4.5 结论句

- `[高频共性]` “综上，针对问题 x，本文给出了……方案/判定方法。”
- `[高频共性]` “综合三问结果，可认为……是当前条件下更优的策略。”

## 4.6 图表引用句

- `[高频共性]` “如图 x 所示，……呈现出明显的……趋势。”
- `[高频共性]` “由表 x 与图 y 结合可知，……”
- `[中频经验]` “图 x 的热力分布进一步验证了……之间的相关结构。”

## 5. 图表语言与学术风格规范

## 5.1 图表引用规则

- 先点名图表编号，再说结论。 `[高频共性]`
- 图表标题应能单独成立，注明对象、时间范围、单位。 `[高频共性]`
- 图表后的正文必须解释“图表支持了什么判断”，不能只说“见图 x”。 `[高频共性]`

## 5.2 描述趋势、差异、变化的常用正式表达

- “呈现上升/下降趋势”
- “波动幅度较大/较小”
- “显著高于/低于”
- “相关性较强/较弱”
- “在某区间内保持稳定”
- “对参数扰动较为敏感/不敏感”

## 5.3 从图表转入分析的推荐方式

- 先讲现象：图里发生了什么。
- 再讲解释：这说明什么。
- 最后回题：这对方案选择、变量判断或结论意味着什么。

## 5.4 容易显得口语化的表达

- “这个结果挺好”
- “效果非常不错”
- “可以明显看出来它很厉害”
- “大概说明了……”

## 5.5 更像正式论文的表达

- “结果表明……”
- “说明该变量对目标函数具有显著影响”
- “可见该方案在当前约束下更具可行性”
- “进一步验证了前述判断”

## 6. 写作常见失误清单

1. **摘要写成流水账。**
   - 只按题号复述，没有方法逻辑和结果信息。

2. **问题分析空泛。**
   - 没说明为什么这样建模，导致正文像模型堆叠。

3. **模型假设不服务后文。**
   - 假设列很多，但正文根本用不上。

4. **结果只报数不解释。**
   - 读者看到了表，却不知道这些数值如何回答题目。

5. **结论没有回题。**
   - 结论只说“本文模型具有现实意义”，没有逐问回收。

6. **图表引用不规范。**
   - 图表编号、标题、单位、正文解释缺一项都会显得粗糙。

7. **语言口语化。**
   - 会明显拉低论文质感。

8. **章节之间断裂。**
   - 问题分析、模型建立、结果分析各说各话。

9. **过度堆公式但解释不足。**
   - 公式很多，变量含义和建模动机却不清楚。

10. **附录替代正文。**
    - 把关键逻辑放进代码或附表，正文只剩结论句。

## 7. 最适合直接写进 `contest-skill` 的写作规则

1. `[高频共性]` 摘要默认按“任务 -> 方法 -> 结果 -> 特点”四段式组织。
2. `[高频共性]` 问题分析必须写出题型判断和题间依赖，不能只写背景套话。
3. `[高频共性]` 每问正文默认采用“模型定义 -> 求解 -> 结果 -> 回题”闭环。
4. `[高频共性]` 图表一律先编号引用，再解释结论，禁止“裸图”。
5. `[高频共性]` 结果分析必须落回题目语言，不只报数值。
6. `[高频共性]` 结论必须按问题顺序回收，不要把摘要改写一遍。
7. `[中频经验]` 统计/诊断型论文应补显著性、残差、ICC、AIC/BIC 等写法模板。
8. `[中频经验]` 变量说明宁精不滥，只列核心记号。
9. `[中频经验]` 优缺点要贴题写，优点写“为什么有效”，缺点写“在什么条件下失效”。
10. `[低频个例]` 深度学习、元学习、复杂算法可以写进摘要，但必须配可解释结果和验证，不可只作“高级感”装饰。
