# Output Contracts

Serves `SKILL.md` Steps 0-9: define what each key stage must output so the workflow does not stop at vague prose.

Call this file when:

- 你知道该做什么，但还不够清楚“做完后该交什么”。
- 你需要把阶段产出压成表格、清单、固定标题块或决策卡片。
- 你怀疑当前回答只有思路，没有交付物。

## Hard Rule

- 面向比赛交付的内容默认中文。
- 启动、单问推进、中期复盘、最终审查、救火纠偏这些阶段，必须结构化输出。
- 如果这些阶段只给自由文字，视为未完成。

## 1. 关键阶段的最小输出格式

### 启动阶段

必须输出：

1. 全题拆题表
2. 依赖链说明
3. 高风险结构清单
4. 全题路线图
5. 前 4 小时最低交付物

最小字段：

| 问题 | 目标 | 题型 | 依赖 | 最终输出对象 | 高风险结构 |
| --- | --- | --- | --- | --- | --- |

### 单问推进阶段

必须输出：

1. 问题定义卡
2. 路线决策卡
3. 预处理与变量设计卡
4. 执行计划
5. 第一版 final answer artifact 草案
6. schema 对齐说明（若题目有 workbook、附件矩阵或固定结果表）

最小字段：

| 项目 | 最少内容 |
| --- | --- |
| 问题定义 | 目标、题型、依赖、最终输出对象 |
| 路线决策 | 主方案、是否需要备选、风险、验证 |
| 预处理 | 粒度、主键、时间轴、特征、proxy 说明 |
| 执行 | 输入、脚本、baseline、首个中间表、首个结果表 |
| 收口 | final answer artifact 草案、结论句 |
| schema 对齐 | 引用的 schema 文件，或至少锁定的表头、主键、时间粒度 |

### 中期复盘阶段

必须输出：

1. 当前阶段判断
2. 已完成/未完成交付物表
3. 每问收口状态
4. 最危险未收口问题
5. 接下来 2-3 小时推进清单

最小字段：

| 问题 | 当前状态 | 已有交付物 | 缺口 | 下一步 | 风险级别 |
| --- | --- | --- | --- | --- | --- |

### 写作冲刺阶段

必须输出：

1. 章节结构或外部格式说明
2. 摘要素材池
3. 图表与正文映射
4. 每问结果块
5. 每问结论句
6. schema / 最终结果表与正文映射

最小字段：

| 问题 | 结果表/图 | 对应 schema/最终表 | 关键结果 | 回题句 | 放正文还是附录 |
| --- | --- | --- | --- | --- | --- |

### 最终审查阶段

必须输出：

1. P0/P1/P2 缺口清单
2. 修补动作
3. 必删内容
4. 最后补漏顺序

最小字段：

| 优先级 | 缺口 | 为什么影响交卷 | 修补动作 | 是否已完成 |
| --- | --- | --- | --- | --- |

### 救火纠偏阶段

必须输出：

1. 当前失败点
2. 删什么
3. 保什么
4. 先补什么
5. 简化路线
6. 接下来 2/4/6 小时动作清单

| 项目 | 内容 |
| --- | --- |
| 失败点 |  |

| 保留资产 |  |
| 优先补的交付物 |  |

### schema / template 落点规则

- 题目给 workbook、附件矩阵、命名结果文件或固定粒度表时，单问推进阶段必须写明对应的 schema 文件；没有专属 schema 时，至少注明已按 `assets/xlsx-schema/generic_answer_sheet_rules.md` 锁定表头。
- 写作冲刺阶段必须说明“正文引用的是 schema 中哪些最终答卷支撑列”，不能只围绕中间分析列写结论。

## 3. 哪些情形必须结构化输出

这些地方必须用表格、编号清单或固定标题块：

- 全题拆题
- 逐问题型判断
- 主方案/备选方案决策
- 预处理设计
- 数据初检记录
- 执行计划
- 执行进度记录
- 验证计划
- final answer artifact
- 中期复盘
- 最终审查
- 救火纠偏

这些地方可以在结构化输出之后补自然段：

- 模型原理解释
- 图表现象解读
- 结果分析段
- 结论段

如果以下场景只给自由文字，视为未完成：

- 启动阶段
- 单问推进阶段
- 中期复盘阶段
- 最终审查阶段
- 救火纠偏阶段

## 4. 语言规则

- 比赛论文、答卷、表格标题、图注、阶段汇报、结论段默认中文。
- 代码注释可中英混合，但正文说明优先中文。
- 模板示例默认用中文字段名。
- 除非用户明确要求英文，否则不要输出英文版段落或英文审查意见。
